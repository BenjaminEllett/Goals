Goals - Set, measure and achieve

Overview

Goals is a web site which helps users set and achieve goals. It does this by helping them measure their progress and learn from setbacks. It also helps them prioritize goals because no one can work towards 10 things at once.

Each goal has several parts. First, each goal has a name and optional description. Second, it contains an optional list of lessons learned. Lessons learned are things the user learned while trying to achieve a goal. Users periodically review them so they remember what they learned and so they can reach their goals in less time. Finally, a goal has one or more metrics. Metrics are very important because they let a person know how they are doing.

Metrics allow individuals to measure their progress towards a goal. Metrics have a name, an optional description, a type (number, time, Yes/No, etc.), a target value, and information about how they are collected and calculated. Here is an example. Suppose an individual wants to reach a heathy weight. The individual decides to measure their weight each week on Sunday morning. Here is the metric the individual creates:

Metric Name Weight (lbs)
Type Whole number (non-negative integer)
Target 170

The individual can easily create this metric with very little work. If the individual wants to include a lot of information and have the site remind him to check the metric weekly, the individual can create a metric with these properties:

Metric Name Weight (lbs)
Description My body's weight in pounds (lbs). It's measure every Sunday night.
Type Whole number (non-negative integer)
Measurement Frequency Weekly
Target 170

The main reason there are multiple ways to create a metric is ease of use. Requiring users to fill out every field makes the program harder to use. It also requires more effort and users might not have the time, energy, desire or need to fill out the optional metric fields.

Users will occasionally not enter metrics. It can be because they forget, they didn't have time or a different reason. This is OK. The Goals web site handles missing metrics. For example, a user forgets to record some measurements during a week. The metric is a weekly metric and records the average week's measurements. The metric will only use the measurements entered to calculate the average. If the user records no measurements, that is also OK. The metric's weekly value will be N/A and charts will exclude weeks with no measurements.

Features

Here are the features the first version of the web site will have:

  • Each user has an account
  • Users can only see data in their account

  • A user's account contains the following:

    • A link to the user's Azure Active Directory account
    • The user's web site settings
    • A list of the user's goals

  • A goals has the following properties:

    • A name
    • An optional description
    • An optional list of associated metrics

    • An optional list of lessons learned. These are notes which explain what the user learned while working toward the goal. For example, if a user's goal is to get to a healthy weight, here are some lessons the user might have learned:

      • I should only buy a small bags of cookies or chips at the super market because I tend eat the entire bag after I get home.
      • I noticed eating plane fat free yogurt, vegetables, fruit and drinking water reduces the urge to over eat.
    • An optional list of notes.
  • User can optionally ask Goals to show one lessoned learned each day. This is helpful because it reminds users of things they may have forgotten. It also helps them avoid past mistakes.
  • A metric has the following properties:
    • A name
    • An optional description
    • A type. Here are the allowed types:
      • Whole number (non-negative integer)
      • Number (integer)
      • Number (decimal)
      • Number (percent)
      • Boolean (true/false)
      • Yes / No
      • Completed / Not completed
      • Time (hours, minutes, seconds)
    • A target value. This is the value of the metric when the user achieves their goal. For example, a user's goal is to lose 10 pounds; the user's weight is 180lbs; and the metric is the user's weight. The target value would be 170lbs in this case.

    • An optional measurement frequency. Here are the supported values:

      • Daily
      • Certain days of the week
      • Weekly
      • Monthly
      • Any time

      There will be a page where users can go to see the metrics which need measurement values. Basically, the site will determine which metrics are ready to be entered and then it will show them on the page. Metrics which have been entered will be removed from the page. Metrics which are never entered will be removed 2 days after the metric's due date.

    • An optional metric period. The measurement frequency and metric period sound similar but have different purposes. The measurement frequency determines how often a user can make measurements. The metric period determines which measurements are converted to a metric value using the metric function. Sum, average, and count are examples of metric functions. Here is an example. If a user wants to measure how much time he spend exercising each week, he would create the following metric:

      Metric Name Exercise time
      Type Time
      Measurement Frequency Any time
      Metric Period Weekly - The week starts on Monday and ends on Sunday
      Metric Aggregation Function Sum

      The value of the metric for a particular week would be the sum of all of the measurements during a particular week. For example, a user exercises 2 hours on Monday, 1.5 hours the next day, and 0.5 hours on Friday. The metric's value would be 4 hours because the metric's period is a week, its aggregation function is sum and all 3 measurements were made during the same week. For some metrics, the measurement frequency and metric period are the same. For example, a metric which measures a person's weight each week might set them the same because the person only measures their weight once each week.

    • An optional metric function. This is only required if more than 1 measurement can be measured during a measurement period. Here is a list of the valid values:
      • Sum
      • Average
      • Count
    • A list of zero or more measurements. Each measurement consists of a value and the instant (date and time) it was recorded.
    • A metric's current value. This can be several things.
      1. If the metric has no measurements during the metric period, it's value is N/A (not available)
      2. If the metric only has one measurement, it's the measurement's value.
      3. If the metric has multiple measurements, it's value is computed by passing all of the measurements to the the metric function.
  • Users can create, update and delete goals
  • Users can create, update and delete metrics
  • Users can create, update and delete metric values. The reason is users sometimes make mistakes and need to correct them.
  • User can visualize their progress using graphs

User Interface Design

The user interface will consist of several pages:

  • Main page - This is displayed when the user first goes to the web site and is not logged in. It states what the site does and has a Sign In button in the upper right corner.
  • Main signed in user page - This page has two parts. First, it shows the metrics which need measurements. User can enter metric values here. Second, it will show all of the user's goals and metrics in priority order. Users can add, update, and delete goals and metrics. Clicking on a goal goes into the goal's page. Clicking on a metric goes to the metric's page.
  • Goal page - This page shows all of a goal's properties including name, description, metrics, lessons learned and notes. Users can change the goal's properties. They can also manage notes, lesson's learned, and metrics. Clicking on a metric goes to the metric's page.
  • Metric page - This page shows a metric's properties including name, type, measurement frequency, metric period, metric function, measurements, and current value. Users can change a metric's properties. Users can also add, update and delete measurements.
